# 提示词模板

> 按需填写，不需要的块直接删掉。

---

## 完整版模板

```
# 角色
你是一个 [领域] 的 [身份]，专注于 [核心职责]。

# 任务
[用一句话说清楚要做什么]

# 输入
[描述输入的格式和内容，或直接在这里放占位符]

# 规则
1. [最重要的约束]
2. [次重要的约束]
3. [其他约束...]

# 边界处理
- 如果 [异常情况A]，则 [处理方式A]
- 如果 [异常情况B]，则 [处理方式B]
- 信息不足时，[统一的 Fallback 行为]

# 输出格式
[描述期望的输出结构，或给一个具体示例]

# 示例（可选）
输入：[示例输入]
输出：[示例输出]
```

## 场景化快速模板

### 场景一：对话助手（System Prompt）

```
你是一个 [领域] 助手，专注于帮助用户解决 [具体问题类型]。

规则：
1. 始终用 [语言] 回答；
2. 回答控制在 [长度要求]；
3. [其他行为要求]；
4. 如果问题超出 [边界]，[处理方式]。
```

---

### 场景二：信息提取（结构化输出）

```
从以下文本中提取 [目标信息]。

提取规则：
- [字段1]：[类型]，[取值范围或格式]；无法判断填 null；
- [字段2]：[类型]，[取值范围或格式]；无法判断填 null；
- [字段3]：[类型]，[取值范围或格式]；无法判断填 null。

如果文本中没有明确提及某项信息，对应字段填 null，不要猜测。

文本：
{input}
```

### 场景三：RAG 问答

```
你是一个严谨的问答助手。请严格基于以下【参考文档】回答用户问题。

规则：
- 只使用参考文档中明确提及的信息作答；
- 禁止推测或补充文档中未提及的内容；
- 如果文档信息不足，回答："根据现有文档无法确定，建议查阅原始资料"；
- [其他特定规则]。

【参考文档】
{context}

【用户问题】
{question}
```

### 场景五：分类 / 判断

```
判断以下内容属于哪个类别。

类别定义：
- [类别A]：[定义和典型特征]
- [类别B]：[定义和典型特征]
- [类别C]：[定义和典型特征]

规则：
- 只能从以上类别中选择一个；
- 如果无法判断，回答 "无法分类" 并说明原因；
- 不要解释，直接给出类别名称。

```

## 填写备忘

| 位置 | 填什么 | 常见错误 |
|------|--------|----------|
| 角色 | 具体的身份+领域，别写"AI助手" | 太泛化，模型行为不可控 |
| 任务 | 一句话，动词开头 | 写了两件事，模型不知道优先做哪个 |
| 规则 | 用编号列表，每条只说一件事 | 大段散文，模型容易漏掉某条 |
| 边界处理 | 写清楚异常情况的处理方式 | 只写正常流程，遇到异常乱编 |
| 输出格式 | 给一个具体示例，比描述更可靠 | 只说"返回JSON"，字段定义不清 |
